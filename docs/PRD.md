#  커피 주문 앱

## 1. 프로젝트 개요

### 1.1 프로젝트 명

### 1.2 프로젝트 목적
사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스텍 웹 앱

### 1.3 개발 범위
- 주문하기 화면 (메뉴 선택 및 장바구니 기능)
- 관리자 화면 (재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능

## 3. 기술 스텍
- 프런트엔드: HTML, CSS, 리엑트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL  

## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음

## 4. 프런트엔드 UI 요구사항

### 4.1 주문하기 화면

#### 4.1.1 화면 구성
주문하기 화면은 사용자가 커피 메뉴를 선택하고 주문할 수 있는 메인 화면입니다.

#### 4.1.2 레이아웃 구조
- **헤더 영역**
  - 앱 로고/제목
  - 관리자 화면으로 이동하는 버튼 (선택사항)
  
- **메인 콘텐츠 영역**
  - 커피 메뉴 목록을 카드 형태로 표시
  - 각 메뉴 카드에는 다음 정보 포함:
    - 메뉴 이미지 (선택사항)
    - 메뉴 이름
    - 가격
    - 재고 상태 표시 (품절 여부)
    - 장바구니 추가 버튼
  
- **장바구니 영역**
  - 화면 하단 또는 사이드에 고정
  - 선택한 메뉴 목록 표시
  - 각 항목별 수량 조절 버튼 (증가/감소)
  - 항목 삭제 버튼
  - 총 주문 금액 표시
  - 주문하기 버튼

#### 4.1.3 주요 기능

**메뉴 선택 기능**
- 메뉴 목록을 그리드 또는 리스트 형태로 표시
- 재고가 없는 메뉴는 비활성화 또는 품절 표시
- 메뉴 카드 클릭 또는 장바구니 추가 버튼으로 메뉴 선택

**장바구니 기능**
- 선택한 메뉴를 장바구니에 추가
- 장바구니 내 메뉴 수량 조절 (증가/감소)
- 장바구니 내 메뉴 삭제
- 장바구니 총 금액 실시간 계산 및 표시
- 장바구니가 비어있을 때 안내 메시지 표시

**주문 기능**
- 주문하기 버튼 클릭 시 주문 확인
- 주문 완료 후 장바구니 초기화
- 주문 완료 메시지 표시 (선택사항)

#### 4.1.4 UI/UX 요구사항

**반응형 디자인**
- 모바일, 태블릿, 데스크톱 환경에서 사용 가능
- 모바일에서는 장바구니를 하단 또는 모달로 표시
- 데스크톱에서는 장바구니를 사이드바로 표시 가능

**상태 표시**
- 재고 부족/품절 메뉴는 시각적으로 구분
- 장바구니에 추가된 메뉴는 시각적 피드백 제공
- 로딩 상태 표시 (메뉴 목록 로딩 중)
- 에러 상태 표시 (주문 실패 시)

**인터랙션**
- 버튼 클릭 시 명확한 피드백 (호버, 클릭 효과)
- 장바구니 항목 추가/삭제 시 애니메이션 효과 (선택사항)
- 부드러운 스크롤 및 전환 효과

#### 4.1.5 데이터 표시

**메뉴 정보**
- 메뉴 ID
- 메뉴 이름
- 가격
- 재고 수량
- 메뉴 이미지 URL (선택사항)

**장바구니 정보**
- 선택한 메뉴 목록
- 각 메뉴별 수량
- 각 메뉴별 소계
- 전체 주문 총액

#### 4.1.6 에러 처리
- 메뉴 목록 로딩 실패 시 에러 메시지 표시
- 주문 실패 시 에러 메시지 표시 및 재시도 옵션 제공
- 네트워크 오류 시 적절한 안내 메시지

#### 4.1.7 기술적 요구사항
- React 컴포넌트 기반 구조
- 상태 관리 (useState, useContext 또는 간단한 상태 관리)
- API 통신을 통한 메뉴 데이터 조회
- API 통신을 통한 주문 데이터 전송
- 로컬 상태로 장바구니 관리 (페이지 새로고침 전까지 유지)
